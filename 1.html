0Claude 요금제 — 어느 걸 결제해야 하나

이 강좌는 Claude Pro 플랜을 기준으로 진행한다.

플랜월 요금Claude Code이 강좌 적합도
Free$0제한적❌ 실습 중 한도 초과 가능
Pro$20 (월간)포함✅ 이 강좌 권장
Max 5x$100~포함 (Pro의 5배)✅ 여유롭게 사용하고 싶은 분

결제 상태 확인하기

  1. claude.ai 로그인
  2. 좌측 하단 프로필 아이콘 → 설정
  3. 좌측 메뉴 결제 클릭 → 현재 플랜 확인
01초기 세팅

VS Code → Claude Code 순서로 설치한다. 10분이면 끝난다.

VS Code 설치

  1. code.visualstudio.com 접속 → OS에 맞는 다운로드 버튼 클릭
  2. Windows: VSCodeSetup-*.exe 실행 → "Add to PATH" 체크 확인 후 설치
    macOS: .zip 압축 해제 → Visual Studio Code.app을 응용프로그램 폴더로 드래그
  3. VS Code 실행 → Extensions 아이콘 → Korean 검색 → 한국어 팩 설치 후 재시작
  4. 메뉴 보기 → 터미널 (단축키 Ctrl + `) 로 터미널 열리는지 확인

Claude Code 설치

VS Code 내장 터미널을 열고 아래 명령어를 붙여넣는다.

PowerShell · 설치
PS> irm https://claude.ai/install.ps1 | iex
⚠️ "irm은 인식되지 않는 이름" 오류가 나면

CMD가 아닌 PowerShell에서 실행해야 한다. 시작 메뉴에서 "Windows PowerShell"을 찾아 실행한다.

zsh · 설치
$ curl -fsSL https://claude.ai/install.sh | bash

설치 확인

설치 확인
$ claude --version
# 예) 2.x.x (Claude Code) 가 나오면 성공
⚠️ "claude: command not found"가 나오면

터미널을 닫고 새로 열어본다. 그래도 안 되면 아래 두 줄을 실행한다.

macOS PATH 등록
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc

로그인과 첫 실행

  1. VS Code 터미널에서 claude 입력 후 Enter
  2. 브라우저가 열리면 Claude.ai 계정으로 로그인
  3. 터미널에 > 프롬프트가 뜨면 성공
  4. 종료: /exit 또는 Ctrl + C
💡 이제부터는 이 터미널이 작업공간

강의 내내 VS Code 터미널에서 claude를 실행해 사용한다. 한 번 로그인하면 유지된다.

02클로드와 대화하는 법

프롬프트 기초 — 잘 시키는 3가지 원칙

원칙나쁜 예좋은 예
구체적으로"엑셀 정리해줘""A열 이름 기준 오름차순 정렬하고 빈 행 지워줘"
결과물 명시"문서 만들어줘""report.docx 파일로 저장해줘"
조건 추가"요약해줘""3줄 이내로 요약하고 핵심 숫자는 굵게 표시해줘"

자주 쓰는 명령어

명령어역할
/help사용 가능한 명령어 목록 확인
/clear대화 내역 초기화 (새 작업 시작)
/exitClaude Code 종료
Ctrl + C현재 작업 중단
↑ 방향키이전에 입력한 명령어 불러오기
ℹ️ 권한 허용 팝업이 뜨면

파일을 만들거나 수정할 때 "허용하시겠습니까?" 팝업이 뜬다. 실습 중에는 허용(Allow)을 누르면 된다.

03GitHub 준비

GitHub는 만든 파일을 저장하고 인터넷에 올리는 창고다. 2주차 배포 실습에 반드시 필요하다.

GitHub 가입 · 저장소 만들기

  1. github.com 접속 → Sign up → 이메일·비밀번호·사용자명 입력 → 이메일 인증
  2. 로그인 후 우상단 + → New repository
  3. 저장소 이름 입력 (예: my-landing) → Public 선택 → Create repository
💡 사용자명이 URL에 들어간다

나중에 배포 URL이 https://저장소이름.vercel.app 형태가 된다. 기억하기 쉬운 영문 소문자로 정한다.

ℹ️ Public vs Private

Vercel 무료 배포를 하려면 Public으로 설정해야 한다.

04Vercel 연결하기

Vercel은 GitHub에 올린 파일을 자동으로 인터넷 주소로 만들어주는 서비스다. 무료 플랜으로 충분하다.

  1. vercel.com 접속 → Continue with GitHub으로 로그인
  2. Vercel이 GitHub 접근 권한 요청 → Authorize 클릭
  3. Vercel 대시보드가 열리면 연결 완료
💡 반드시 GitHub으로 로그인해야 한다

이메일로 따로 가입하지 말고 Continue with GitHub으로 가입해야 저장소와 자동 연결된다.

배포 흐름 미리 보기

지금은 구조만 이해한다. 실제 배포는 2주차에 직접 실습한다.

단계하는 일소요 시간
① GitHub 업로드완성된 index.html을 저장소에 올린다1분
② Vercel ImportVercel에서 저장소를 선택해 가져온다1분
③ DeployDeploy 버튼 클릭 → 자동 빌드1~2분
④ URL 발급https://xxx.vercel.app 주소가 생긴다즉시
—트러블슈팅

claude 명령어가 안 된다

터미널을 닫고 새로 열기

설치 직후에는 현재 터미널에 PATH가 반영되지 않는다. 터미널을 완전히 닫고 새로 열면 대부분 해결된다.

로그인 화면이 브라우저에서 안 열린다

기본 브라우저 설정 확인

Windows에서 기본 브라우저가 없으면 브라우저가 안 열릴 수 있다. Chrome 또는 Edge를 기본 브라우저로 설정한 뒤 다시 시도한다.

한국어로 지시했는데 영어로 답한다

한국어로 답해달라고 명시

프롬프트 끝에 "한국어로 답해줘"라고 추가하면 된다. 한 번 말해두면 같은 세션에서 유지된다.

Vercel에 GitHub 저장소가 보이지 않는다

GitHub 권한 설정 확인

Vercel 대시보드 → Settings → Git → Configure GitHub App을 클릭해 접근 허용할 저장소를 추가한다.
