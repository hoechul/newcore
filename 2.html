01HTML이 뭔가

HTML은 웹페이지의 뼈대다. 직접 쓸 필요는 없다 — Claude Code가 다 만들어 준다. 하지만 구조를 눈으로 보면 수정 요청을 훨씬 정확하게 할 수 있다.

페이지 뼈대

HTML · 기본 구조
<!DOCTYPE html>
<html lang="ko">
<head>
  <!-- 브라우저 탭 제목, 폰트, 스타일 -->
  <title>내 페이지 제목</title>
</head>
<body>
  <!-- 실제로 보이는 내용 전부 여기 -->
  <h1>안녕하세요</h1>
  <p>저는 홍길동입니다.</p>
</body>
</html>

자주 쓰는 태그

태그역할예시
<h1>~<h6>제목 (숫자 클수록 작음)페이지 대제목, 섹션 제목
<p>문단 텍스트본문, 설명
<a>링크버튼, 외부 URL
<img>이미지프로필 사진, 로고
<div>레이아웃 묶음섹션, 카드, 그리드
ℹ️ 외울 필요 없다

이 표는 Claude Code가 만든 코드를 볼 때 참고용이다. 수정할 때는 클로드에게 말로 요청하면 된다.

02클로드로 페이지 만들기

빈 폴더 열고 claude 실행

  1. 바탕화면에 my-landing 폴더를 새로 만든다
  2. VS Code에서 파일 → 폴더 열기로 my-landing을 연다
  3. 내장 터미널(Ctrl + `)을 열고 claude 입력 후 Enter

첫 프롬프트 — 페이지 생성

클로드코드에 입력
한 페이지짜리 개인 소개 랜딩페이지를 index.html 파일 하나로 만들어줘.

이름: [본인 이름]
직함 또는 하는 일: [예: 스마트스토어 운영자]
한 줄 소개: [예: 10년 경력 의류 셀러, 패션 트렌드를 가장 빠르게 전달합니다]
연락처: [이메일 또는 전화번호]
색상 분위기: [예: 어둡고 세련된 느낌]

섹션: 상단 히어로(이름·소개·버튼), 소개 섹션, 연락처 섹션.
Google Fonts 사용하고, CSS는 파일 안에 같이 써줘.

브라우저에서 미리보기

Windows
PS> start index.html
macOS
$ open index.html

반복 수정 — 말로 요청하면 끝

원하는 변경클로드에게 이렇게 말한다
글자 바꾸기"히어로 섹션 제목을 '안녕하세요, 홍길동입니다'로 바꿔줘"
색상 변경"버튼 색깔을 #534AB7 보라색으로 바꿔줘"
섹션 추가"소개 아래에 서비스 3가지를 카드로 보여주는 섹션 추가해줘"
모바일 대응"스마트폰에서도 잘 보이도록 반응형으로 수정해줘"
폰트 변경"본문 폰트를 Noto Sans KR로 바꿔줘"

이미지 넣기

사진이나 로고를 넣으려면 이미지 파일을 my-landing 폴더 안에 복사해 넣고 클로드에게 알린다.

클로드코드에 입력
폴더에 profile.jpg 파일을 넣었어. 히어로 섹션에 원형 프로필 사진으로 넣어줘. 크기는 150×150px.
⚠️ 이미지 파일명에 한글 쓰지 않기

내사진.jpg 대신 profile.jpg처럼 영문으로 저장해야 배포 후에도 정상 표시된다.

03GitHub에 올리기

완성된 index.html을 GitHub 저장소에 올려야 Vercel이 읽어서 배포할 수 있다.

파일 업로드 (drag & drop)

  1. github.com 로그인 → 저장소 페이지 이동
  2. Add file → Upload files 클릭
  3. index.html과 이미지 파일들을 드래그해서 업로드 창에 놓는다
  4. 하단 Commit changes 클릭

터미널로 올리기 (더 빠른 방법)

터미널 · 최초 1회
# my-landing 폴더 안에서 실행
$ git init
$ git remote add origin https://github.com/내아이디/my-landing.git
$ git add .
$ git commit -m "첫 배포"
$ git push -u origin main
터미널 · 수정 후 업로드 (매번)
$ git add .
$ git commit -m "내용 수정"
$ git push
04Vercel 배포
1
Step 01
vercel.com → GitHub으로 로그인
vercel.com 접속 → Continue with GitHub으로 로그인
2
Step 02
Add New Project → 저장소 선택
Add New → Project 클릭 → my-landing 저장소 찾아 Import
3
Step 03
설정 그대로 → Deploy 클릭
Framework: Other, Root Directory: ./ 그대로 두고 Deploy 클릭
🎉
완료
URL 발급 — 누구나 접속 가능
배포 URL 예시
https://my-landing.vercel.app
💡 이후 수정은 자동 반영

GitHub에 파일을 다시 올리면 Vercel이 자동 감지해 30초~1분 안에 최신 버전으로 반영된다.

05도메인 연결 (선택)

xxx.vercel.app 대신 독자 도메인을 쓰고 싶으면 연결할 수 있다.

단계내용비용
도메인 구매가비아·네임칩 등에서 원하는 도메인 구매연 1~3만원
Vercel에 추가프로젝트 설정 → Domains → 도메인 입력무료
DNS 연결가비아에서 Vercel이 알려주는 값 복사해 넣기무료
HTTPS 자동Vercel이 SSL 인증서도 자동 발급무료
ℹ️ 오늘 꼭 안 해도 된다

Vercel URL로도 충분히 공유 가능하다. 나중에 명함에 넣고 싶을 때 연결해도 늦지 않는다.

—트러블슈팅

이미지가 배포 후 안 보인다

이미지 파일이 GitHub에 같이 올라가지 않았다

GitHub 저장소에서 이미지 파일이 있는지 확인한다. 없으면 drag & drop으로 이미지를 추가 업로드하고 Commit한다.

Vercel 배포 후 빈 화면이 나온다

파일명이 index.html이 아니다

Vercel은 기본적으로 index.html을 찾는다. 파일명을 정확히 index.html로 저장했는지 확인한다.

git push 시 비밀번호를 물어본다

Personal Access Token을 써야 한다

GitHub → Settings → Developer settings → Personal access tokens → Generate new token. 생성한 토큰을 비밀번호 칸에 붙여넣는다.
